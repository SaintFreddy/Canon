## Source: Chat-Native Release Plan — R1 Transcript Chat

## 5. Release 1 — Transcript Chat

### Product promise
The product feels like familiar AI chat, but each answer is already a bounded run with inspectable source grounding and a frozen context snapshot.

### Why this release exists
Adoption. This is the low-friction wedge.

### What ships
- Conversation composer and transcript canvas
- File and URL attachments
- Thread library
- Inline citations and source chips
- “What was used” compact drawer on each assistant answer
- Basic regenerate / rerun
- Basic artifact outputs from answers
- Thread resume packet

### What becomes explicit for the first time
- Runs exist, even if the user still experiences them as chat turns
- Context is snapshotted per answer
- Sources have identity independent of the thread
- Messages are outputs, not truth

### Primitives required

**Shared / environment**
- Thread / Line
- Run
- Source Reference
- Artifact
- Context Pack snapshot
- Proof Bundle summary
- Provenance Graph basics

**Engine**
- Execution Unit
- Compiled Context Package
- Evidence Item / Evidence Set
- Model Invocation normalization
- Event stream
- Minimal validation hooks

**Chat-domain**
- Message Block
- Conversation Surface
- Source Chip
- Resume Packet

### SDK packages forced to mature
- `models`
- `orchestration` (minimal interactive path)
- `artifacts`
- `provenance`
- `reference` adapters for basic chat/runtime

### Exit criteria
- Every assistant answer has a run ID, frozen context snapshot, and source list
- Threads survive model swaps and provider changes
- Basic citations resolve to source regions
- The system does not require a hidden provider-managed transcript to continue the conversation

### What is deliberately not shipped yet
- Editable packs
- Branch graph
- Artifact-centered workspace
- Protocols / applets
- Formal commissioning
