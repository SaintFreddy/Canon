## Source: All-Prims — What Does Not Belong in the Environment

### 15. WHAT DOES NOT BELONG IN THE ENVIRONMENT PRIMITIVE SET
The following should not be promoted to environment primitives.

#### 15.1 Raw prompt blob

A prompt may be input, but it is not a stable environment object.

#### 15.2 Transcript as ground truth

Continuity is useful, but the environment cannot collapse consequence back into chat history.

#### 15.3 Decorative agent cards

Agent theater is not a primitive.

#### 15.4 Generic one-click save

Writeback lanes must remain distinct.

#### 15.5 Unstructured skills.md file as the reuse primitive

A skills.md file may be an authoring aid, but the environment reuse primitive must be Protocol, not prose.

#### 15.6 Mandatory visual workflow graph as the only UX

Some domains may use graph views, but the environment must not force all work into low-code node editing.

#### 15.7 File tree as the semantic center of code work

Useful as a projection in code apps, but not a shared environment primitive.

#### 15.8 One undifferentiated memory bucket

Memory and Canon must remain distinct.

#### 15.9 App-private ontologies that duplicate shared objects

An app may add domain primitives, but it must not replace Run, Artifact, Branch, Proof, Writeback, or Scope with private incompatible substitutes.
